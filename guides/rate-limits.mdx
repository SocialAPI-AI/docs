## How credits work

Every API request consumes **credits** from your monthly allowance. Most endpoints cost 1 credit. Write operations (replies, sending DMs) cost 2 credits.

Credits reset at the start of each billing period — aligned with your Stripe subscription cycle. Free-tier users get a rolling 30-day window from account creation.

## Plan credits

| Plan | Credits / month | Price |
|---|---|---|
| Free | 100 | $0 |
| Starter | 500 | $29 |
| Pro | 5,000 | $149 |
| Business | 50,000 | $799 |
| Enterprise | Unlimited | Custom |

Credits are the **only** differentiator between plans. All plans have access to all platforms, DMs, and features.

Upgrade at [app.social-api.ai](https://app.social-api.ai) → **Billing**.

## Credit costs per endpoint

| Operation | Cost | Examples |
|---|---|---|
| Read | 1 credit | List comments, fetch DMs, list posts |
| Write | 2 credits | Reply to comment, send DM |

## When you run out

The API returns HTTP `429` with:

```json
{
  "error": "monthly credit limit exceeded for your plan",
  "code": "rate_limit_exceeded"
}
```

Your credits reset at the start of your next billing period. To get more credits immediately, upgrade your plan.

```bash
curl https://api.social-api.ai/v1/usage \
  -H "Authorization: Bearer $SOCAPI_KEY"
```

```json
{
  "credits_used": 142,
  "credits_limit": 500,
  "period_start": "2026-02-15T00:00:00Z",
  "period_end": "2026-03-15T00:00:00Z"
}
```

`credits_limit` is `-1` for Enterprise (unlimited).

## Platform rate limits

Platforms (Instagram, Facebook, etc.) have their own rate limits independent of your credit balance. When a platform rejects a request due to its own limits, you receive:

```json
{
  "error": "platform rate limit exceeded",
  "code": "platform_rate_limit"
}
```

This does **not** consume credits. Retry after a short wait (typically a few minutes).

## Handling 429s

```python
import time
import requests

def call_with_retry(url, headers, max_retries=3):
    for attempt in range(max_retries):
        resp = requests.get(url, headers=headers)
        if resp.status_code == 429:
            time.sleep(2 ** attempt)  # exponential backoff
            continue
        return resp
    raise Exception("credit limit exceeded after retries")
```

<Note>
  If the credit enforcement backend is unavailable, the API **fails open** — requests are allowed through rather than blocked. This prevents infrastructure downtime from taking down your integration.
</Note>
