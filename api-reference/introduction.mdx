## Base URL

```
https://api.socapi.ai/v1
```

## Authentication

All endpoints require a Bearer token:

```bash
Authorization: Bearer soc_live_...
```

See [Authentication](/guides/authentication) for details.

## Interaction ID prefixes

SocAPI IDs encode their type in a prefix so you can dispatch replies without a database lookup:

| Prefix | Type |
|---|---|
| `socapi_cmt_` | Comment |
| `socapi_rev_` | Review |
| `socapi_dm_` | Direct message |
| `socapi_mnt_` | Mention |

## Error format

All error responses use the same shape:

```json
{
  "error": "human-readable message",
  "code": "machine_readable_code"
}
```

| HTTP status | Meaning |
|---|---|
| `400` | Invalid request parameters |
| `401` | Invalid or missing API key |
| `403` | Feature not available on your plan |
| `404` | Account or resource not found |
| `429` | Daily rate limit exceeded |
| `501` | Platform does not support this operation |
